## O que esta skill contém

- Framework principal P→E→S→I
- Elevator pitch (60 segundos)
- Estrutura de 7 slides
- Adaptação por audiência
- Extração de evidências por agente

## Quando carregar esta skill

Carregue quando:
- Criar apresentação executiva
- Defender decisões de design com stakeholders
- Comunicar resultados de pesquisa para diferentes audiências
- Criar elevator pitch

---

## Framework Principal — Problema → Evidência → Solução → Impacto

```
1. PROBLEMA (30s)
   → Qual dor real do usuário foi identificada?
   → Qual o custo desse problema para o negócio?

2. EVIDÊNCIA (1min)
   → O que a pesquisa mostrou?
   → O que os dados confirmaram?
   → O que a avaliação heurística revelou?

3. SOLUÇÃO (1min)
   → O que o time propõe e por quê?
   → Quais princípios guiaram a decisão?

4. IMPACTO (30s)
   → Qual métrica vai melhorar? Em quanto tempo?
   → Qual o próximo passo concreto?
```

## Elevator Pitch — 60 segundos

```
"Descobrimos que [usuário específico] tem dificuldade com [problema concreto],
 o que causa [impacto mensurável].

 Nossa pesquisa com [n usuários / dados de X] confirmou que [evidência principal].

 Por isso propomos [solução em uma frase], que vai [resultado esperado]
 medido por [métrica específica].

 O próximo passo é [ação concreta] em [prazo]."
```

## Estrutura de Apresentação Executiva (7 slides)

```
S1 — Headline:           uma frase que resume o projeto inteiro
S2 — O problema:         dor do usuário em linguagem humana + impacto em números
S3 — Como descobrimos:   métodos sem jargão + volume de evidências
S4 — O que encontramos:  top 3 insights + citação de usuário + dado quantitativo
S5 — A solução proposta: recomendação + princípios + o que foi descartado
S6 — Impacto esperado:   métrica que vai mover + prazo + como medir sucesso
S7 — Próximos passos:    ação + responsável + prazo por item
```

## Adaptação por Audiência

```
CEO / C-Level:   direto, ROI, vantagem competitiva, risco — 3-5 min
PM / Stakeholders: colaborativo, trade-offs, KPIs — 10-15 min
Dev Team:        técnico, specs, critérios de aceite — 20-30 min
Cliente externo: consultivo, valor entregue, próximos passos — 15-20 min
```

## Extração de Evidências por Agente

```
De heuristic-evaluator:
  Antes: "H6 - usuário não identifica onde está no fluxo de contratação"
  Após:  "Usuários se perdem durante a contratação — problema em 4 das 7 jornadas"

De ux-research:
  Antes: "Insight #3: usuários com baixa literacia abandonam na documentação"
  Após:  "1 em cada 3 usuários desiste no envio de documentos — não por falta de
          interesse, mas por não entender o que é pedido"

De data-analyst:
  Antes: "Drop-off rate etapa 3: 42%"
  Após:  "Perdemos 42% dos interessados antes da proposta — o equivalente a R$ X/mês"
```

## Checklist de Qualidade

```
□ Problema em linguagem do usuário, não de UX
□ Todas as afirmações têm evidência do time
□ Nenhum jargão técnico sem explicação
□ Impacto quantificado ou estimado
□ Próximos passos concretos com responsável
□ Adaptado para a audiência correta
□ Duração adequada ao contexto
```
