## O que esta skill contém

- Métodos de pesquisa por fase (Descoberta, Definição, Avaliação)
- Templates: persona, journey map, roteiro de entrevista, insight
- Padrões de qualidade para pesquisa

## Quando carregar esta skill

Carregue quando:
- Conduzir pesquisa com usuários (entrevistas, surveys, testes)
- Criar ou revisar personas ou journey maps
- Documentar insights de pesquisa
- Planejar metodologia de UX research

---

## Métodos por Fase

```
Descoberta:  entrevistas com stakeholders, desk research, análise competitiva
Definição:   entrevistas semi-estruturadas, surveys, card sorting, JTBD
Avaliação:   testes de usabilidade, first-click, tree testing, avaliação heurística
```

## Templates Principais

### Persona
```
PERSONA: [Nome]
─────────────────────────────
Perfil: [idade, ocupação, contexto de vida]
Objetivos: [principal, secundário]
Frustrações: [dor principal, dor secundária]
Comportamentos: [padrão comportamental chave]
Citação: "[frase representativa]"
Conforto com tecnologia: [baixo / médio / alto]
Insight chave para o produto: [uma frase]
```

### Journey Map
```
JOURNEY MAP: [Persona] realizando [Tarefa]
─────────────────────────────
Etapas: Descoberta → Consideração → Decisão → Uso → Fidelização

Por etapa:
→ Ações:         o que o usuário faz
→ Pensamentos:   o que ele pensa
→ Sentimentos:   estado emocional (😟 → 😐 → 😊)
→ Pontos de dor: momentos de fricção
→ Oportunidades: intervenções de design
```

### Roteiro de Entrevista
```
Introdução (5 min): consentimento, contexto, aquecimento
Contexto do usuário (10 min): background e comportamento atual
Tópico principal (25 min): perguntas centrais da pesquisa
Reação ao conceito (10 min): se estiver testando estímulo
Encerramento (5 min): algo mais a acrescentar

Perguntas: sempre abertas, comportamentais, situacionais
Ex: "Me leva pelo último momento em que você [comportamento]"
```

### Formato de Insight
```
INSIGHT #[N]
─────────────────────────────
Descoberta: [o que observamos]
Evidência:  [citações ou pontos de dados]
Padrão:     [quantos usuários / com que frequência]
Impacto:    [Alto / Médio / Baixo]
Oportunidade: [implicação de design]
```

## Padrões de Qualidade

```
→ Mínimo 5 participantes para estudos qualitativos
→ Recrutar do ICP real, não por conveniência
→ Separar observações de interpretações
→ Triangular descobertas em pelo menos 2 métodos
→ Sinalizar hipóteses ainda não validadas
```
